1. 2.Программирование на JavaScript: основы

При разработке на языке JavaScript мы часто встречаем понятие EcmaScript или аббревиатуру ES.

* JavaScript — популярное расширение языка EСMAScript, который был создан для обработки действий на компьютерах пользователей.

Название EСMAScript сейчас используется для обозначения версий языка. Самая популярная и актуальная — ES6.

JavaScript встроен почти в любой веб-браузер, и потому доступен почти на каждом вычислительном устройстве.

На первых порах JavaScript кажется простым, потом сложности возникают из-за того, что система не сообщает об ошибках, а интерпретирует их на свой лад.

Так какой же язык JavaScript?

Чтобы в этом разобраться, рассмотрим классификации языков программирования. Давайте уделим время теории и проанализируем различные принципы группирования языков.

1. Языки высокого и низкого уровней

Все устройства получают инструкции, благодаря которым они реализуют определённое действие: открытие, скачивание файла, удаление и так далее. Эти инструкции представлены в виде машинного кода.

Писать программы непосредственно в машинном коде возможно. Однако это довольно утомительно и подвержено ошибкам, так как необходимо управлять отдельными битами и вычислять числовые адреса и константы вручную. По этой причине машинный код практически не используется для написания программ. Для этих целей существуют различные языки программирования.

Разработчик, использующий языки низкого уровня , обращается в своей программе непосредственно к ресурсам компьютера: процессору, памяти, периферийным устройствам. Это позволяет обеспечить высокую скорость работы программ, поскольку, в отличие от высокоуровневых языков, здесь отсутствуют скрытые фрагменты кода, добавляемые автоматически компилятором во время преобразования исходного текста в бинарный код.

Когда программист пишет программу на определённом языке, машина не может её прочитать.

Текстовый файл переводится в машинный код с помощью инструмента, называемого «ассемблер»*.

* *Интерпретаторы, компиляторы, ассемблеры— это утилиты, с помощью которых исходный код транслируется в исполняемый машинный код. В браузере таким интерпретатором является движок.

При написании программ с использованием высокоуровневых языков нет необходимости задумываться об архитектуре компьютера — необходимый уровень абстракции достигается за счёт скрытых от разработчика проверок и функциональности, которая позволяет не задумываться о таких вещах, как выделение памяти под переменные, предоставление пространства и другое.

Языкам высокого уровня свойственна работа с комплексными структурами данных. В большинстве из них интегрирована поддержка строковых типов, объектов, операций файлового ввода-вывода и тому подобное.

Такие языки разработаны для быстроты и удобства использования. Основная черта высокоуровневых языков — это абстракция, то есть введение смысловых конструкций, которые кратко описывают такие структуры данных и операции над ними, описания которых на машинном коде (или любом низкоуровневом языке программирования) очень длинны и сложны для понимания.

Сравните исходные коды выше — у высокоуровневого он значительно короче.

2. Компилируемые и интерпретируемые языки

В компилируемых исходный код преобразуется компилятором в машинный код и записывается в файл, который затем исполняется. Если компилятор найдёт ошибки, то приложение не запустится.

К компилируемым языкам относятся, например, C++ и Java.

Исходный код в интерпретируемых языках исполняется программой-интерпретатором. Код поставляется как есть — интерпретатор его выполняет, даже если есть ошибки.

К интерпретируемым языкам относится, например, JavaScript.

Интерпретируется JavaScript специальным движком — это программа, выполняющая код. Движок может быть реализован с использованием различных подходов: в виде обычного интерпретатора, в виде динамического компилятора (или JIT-компилятора).

V8 — движок в Chrome и Node.js.

3. Типизированные и нетипизированные языки

Языки программирования по типизации принято делить на два больших лагеря — типизированные и нетипизированные (бестиповые).

Типизированные языки (typed) — это те, в которых существуют и используются разные типы данных, например, булевы значения, числа, строки и другие. Над этими данными выполняются соответствующие операции — математические над числами, конкатенация над строками и тд. Это такие языки, как C, Python, Scala, PHP и Lua, JavaScript

Бестиповые (untyped) — это низкоуровневые языки, работающие с машинным кодом (в них обработка данных происходит как операция над цепочкой битов). Представитель этой группы — язык ассемблера.

Исходя из этого, можно выделить ещё две категории языков.

4. Языки со статической и динамической типизацией

Статическая типизация определяется тем, что конечные типы переменных и функций устанавливаются на этапе компиляции. То есть компилятор на 100% уверен, какой тип где находится, и переопределение невозможно.

В динамической типизации все типы выясняются уже во время выполнения программы. То есть можно объявить переменную строкового типа, а затем записать в неё число. Интерпретатор на каждом этапе выполнения будет считывать новое значение и определять его тип.

5. Языки с сильной и слабой типизацией

Сильная типизация не позволяет смешивать в выражениях различные типы и не выполняет автоматические неявные преобразования. Например, нельзя вычесть из строки множество.

Языки со слабой типизацией выполняют множество неявных преобразований автоматически, даже если может произойти потеря точности или преобразование неоднозначно. Это позволяет нам в JavaScript делать подобные действия: “123” + 1, [] + {}, true + 0 и получать конкретный результат, который не приведёт к ошибкам. (Но это не означает, что нужно так делать :))

Резюмируем

JavaScript — это интерпретируемый язык высокого уровня с динамической и слабой типизацией.

Что такое скрипты?

Что такое скрипты и зачем их использовать?

Скриптами называют инструкции-сценарии, написанные на скриптовых языках программирования, другими словами, это код программы.

Первые скриптовые языки начали разрабатываться в 1960-х с целью автоматизации процессов при написании программ. Их возможности были очень примитивными, а основной целью — избавление от повторов нескольких команд. В отличие от компилируемых языков программирования, таких как C, C++, Pascal, их исходный код не преобразовывается в машинный для выполнения процессором, а выполняется с помощью интерпретатора.

Скрипты — это только про JavaScript?

Нет, JavaScript далеко не единственный скриптовый язык, но наиболее используемый в веб-разработке. Изначально JavaScript был создан, чтобы сделать веб-страницы «живыми», добавить динамику в HTML-разметку со стилями. Сегодня JavaScript может выполняться не только в браузере, но и на сервере или на любом другом устройстве, которое имеет специальную программу, о которой мы уже упоминали выше, называющуюся «движком» JavaScript.

Стоит также иметь в виду, что JavaScript (JS) никак не связан с другим известным языком программирования Java.

JS появился в тот момент, когда язык Java был на пике популярности, и создатели JavaScript просто воспользовались его славой и выбрали очень похожее название.

Какими возможностями обладает JS в браузере?

  • Добавлять, изменять содержимое HTML-страницы, модифицировать стили.
  • Реагировать на действия пользователя, такие как нажатия клавиш, щелчки мыши.
  • Отправлять запросы на удалённые сервера, скачивать и загружать файлы.
  • Напишете первый код, используя сайт-песочницу;
  • Получать и устанавливать куки.
  • Запоминать данные на стороне клиента.
  • Запоминать данные на стороне клиента.

Тем не менее JavaScript в браузере не всесилен: у него есть много ограничений, необходимых для безопасности пользователей. Так, например, JS не может получать доступ к чтению/записи произвольных файлов на жёстком диске.

Посмотрите, что станет со страницами, если на какое-то время отключить на них JavaScript. Следуйте инструкциям в зависимости от вашего браузерa.

Типы данных

Представьте, что вы пришли в магазин сотовой связи, чтобы приобрести новый гаджет. Между вами и сотрудником магазина разворачивается следующий диалог:

Вы: Сколько стоит этот смартфон?

Сотрудник: 9 990.

Вы понимаете, что он подразумевает стоимость в рублях, и запоминаете данную информацию.

Если вам понадобятся два таких смартфона, то вы сможете посчитать их стоимость в уме — 19 980.

Сотрудник: Сегодня у нас также проводится акция «Невиданная щедрость». Телефон предыдущей модели даём в подарок!

Вы не можете поверить своим ушам, ведь сказанное им означает, что телефон стоит 0 рублей. Вы решаете купить телефон и получить другой бесплатно.

Сотрудник: Вам нужны для них чехлы?
Вы: Да!

Похожим образом, когда вы выражаете значения в программах, вы выбираете среди различных типов. Если вам нужна математика, то вы выбираете числа, если необходимо напечатать значение на экране, то строку, а когда перед вами стоит выбор — логический тип данных.

Итак, JavaScript определяет 8 встроенных типов данных:

  • null
  • undefined
  • boolean
  • number
  • string
  • symbol
  • object
  • bigint

Все эти данные, кроме object, являются «простыми» или «примитивными» (их ещё называют скалярные).

Примитивные значения— это единица информации, которая представлена непосредственно на нижнем уровне реализации языка. Примитивные типы данных — значения, которые не являются объектами и не имеют методов. Все примитивные значения неизменяемы (immutable). Не имеет смысла говорить об изменчивости, например, значения числа.

Рассмотрим подробнее свойства каждого из примитивных типов данных:

Нулевой (Null) Может принимать только значение null.
Неопределённый (Undefined) Переменная, которой не было присвоено значение, будет иметь значение undefined.
Логический (Boolean) Имеет два значения true или false.
Числовой (Number)
  • Числа могут быть положительными или отрицательными. Кроме того, есть возможность хранить дроби, отделяя целое точкой: 12.3984
  • Очень большие числа записываются с буквой е (экспонентой): 2.993е8 — это 2.993 * 10^8
  • Специального типа для целых чисел в JavaScript нет. Это означает, что при числовых операциях вы можете получить неточное (округлённое) значение.
Строчный (String)
  • Данные записываются в одинарных, двойных или обратных кавычках. Главное, использовать их вместе: 'Что посеешь, то и пожнёшь' или "Делу время, потехе час".
  • Для вывода специальных символов используется обратный слэш.
  • Перевод строки обозначается символом \n.
  • Строки нельзя делить, умножать и вычитать. Однако можно применять оператор +, который будет соединять их друг с другом. "Миру" + "мир".
  • Строки в JavaScript нельзя модифицировать.
bigInt Так как числовой тип данных не может содержать числа длиннее, чем 16 символов, был добавлен тип bigInt, чтобы работать с целыми числами произвольной длины.

Основные конструкции языка (операторы)

Итак, любая программа — это последовательность выполняемых инструкций. Инструкция состоит из выражений, которые, в свою очередь, содержат операторы.

Можно провести аналогию с обычным языком: инструкция — это предложение, выражение — это фразы, а операторы — союзы/пунктуация. То есть операторы связывают операнды (запятая, согласующая части предложения) или применяются к операнду (восклицательный знак в конце предложения).

Операторы могут быть разбиты на категории по количеству требуемых им операндов:

унарные

operator operand;

++a;

operand operator;

a++;
бинарные

operand operator operand;

operand operator operand; a + b; a * b;
тернарные

на данный момент существует только один — условный оператор;

operand operator operand operator operand;

a > b ? a : b

На самом деле существует большая классификация операторов. Но пока мы остановимся на самых часто используемых. По принципу действия выделяют следующие:

  • Операторы присваивания,
  • Операторы сравнения,
  • Операторы отношения,
  • Арифметические операторы.

Прежде чем мы разберём каждый в отдельности, пару слов о переменных. Подробнее об объявлении переменных мы будем говорить чуть позже, но чтобы не запутаться сейчас, запомните несколько ключевых идей:

Переменные — это контейнеры, в которых можно хранить значения. В определённый момент у каждой переменной может быть только одно значение.

Переменные объявляют при помощи var. Если переменную не объявить, то интерпретатор не будет знать, откуда её брать. При этом иногда в консоли вы не будете видеть ошибку. Другие способы объявлений в JavaScript:

let — объявляет локальную переменную в области видимости (то есть в контексте данной функции). Указание значения, так же как и при использовании var, необязательно.

const — объявляет именованную константу*. Значение доступно только для чтения.

*Константы — это переменные, которые не будут подвержены изменениям. Их принято писать заглавными буквами, разделяя нижним подчёркиванием.

Типы операторов

По принципу действия, оказываемого на операнды, выделяют:

Операторы присваивания

Операторы присваивания — это бинарные операторы operand operator operand;

let x; — заранее созданная переменная

x = 10;

x = 'myname';

console.log(x = 100 + 1); // в консоли: 101

Сначала вычисляется значение справа (right-hand side) от оператора, а затем вычисленное значение записывается в операнд, который находится слева (left-hand side) от оператора.

Основной оператор присваивания — это знак равно =.

Слева от оператора присваивания (=) может находиться только операнд, которому можно присвоить значение — переменная или свойство объекта!

Например, если числу присвоить число, то консоль выдаст ошибку:

let x, y;

x = 10;

y = 'hello';

10 = 101; // Uncaught ReferenceError: Invalid left-hand side in assignment

С полным списком операторов присваивания можно ознакомиться здесь.

Любое выражение возвращает значение. Не является исключением выражение, содержащее в себе оператор =.

В примере ниже значение справа записывается в переменную x, а затем возвращается:

let x;

console.log(x = 10);

10

Благодаря этому присваивание можно использовать как часть более сложного выражения:

let a, b;

b = a = 10;

10

ВНИМАНИЕ!

let a = 10; — инструкция, не возвращает значения.

a = 10; — выражение, возвращает значение.

Операторы сравнения

Операторы сравнения — бинарные операторы (operand operator operand), которые, как и другие операторы, возвращают значения.

Это значение имеет логический тип boolean (true или false).

Существует строгое (===) и нестрогое (==) сравнение.

Перед тем как непосредственно выполнить нестрогое сравнение, операнды приводятся к одному типу. Строгие сравнения не изменяют типы данных своих операндов, поэтому они имеют более предсказуемое поведение.

Оператор Оператор
Равно (==) Возвращает true, если операнды равны. Оператор сначала приводит операнды к одному типу, и затем применяет строгое сравнение.
Не равно (!=) Возвращает true, если операнды не равны. Аналогично оператору равенства, перед сравнением приводит операнды к одному типу.
Строго равно (===) Возвращает true, если операнды строго равны и имеют одинаковый тип.
Строго не равно(!==) Возвращает true, если операнды не равны и/или имеют разный тип.
Больше (>) Возвращает true, если операнд слева больше операнда справа.
Больше или равно (>=) Возвращает true, если операнд слева больше или равен операнду справа.
Меньше (<) Возвращает true, если операнд слева меньше операнда справа.
Меньше или равно (<=) Возвращает true, если операнд слева меньше или равен операнду справа.

При использовании данных операторов происходит приведение типов!

Соглашение об именованиях

Перед тем как научимся объявлять переменные, мы изучим, как же их правильно называть.

Правильный выбор имени переменной — одна из самых важных и сложных вещей в программировании. Дело в том, что большую часть времени мы тратим не на изначальное написание кода, а на его развитие. Гораздо проще найти нужные данные, если они правильно помечены, то есть когда переменная названа правильно.

Соглашение о наименованиях (naming convention) — набор правил для выбора последовательности символов, которые будут использоваться для имён переменных и функций.

Существуют следующие нотации:

  • snake_case — пробелы между словами заменяются на нижнее подчеркивание.
  • PascalCase — все слова вместе, первая буква каждого слова — заглавная.
  • camelCase (верблюжья нотация) — преимущественно используется в JS, в частности во встроенных языковых и браузерных функциях (этого правила будем придерживаться в этом модуле).

Важно также придерживаться следующих правил:

  1. Никакого транслита.
  2. Использовать короткие имена только для переменных «местного значения».
  3. Имя переменной должно чётко соответствовать хранимым в ней данным.

Как выбрать имя для переменной (identifier)

1. Всегда начинать с буквы (в особых случаях допустимы также значок $ или нижнее подчеркивание). Хотя само имя может содержать последовательность букв, цифр и нижнего подчеркивания.

2. Учитывать, что JavaScript чувствителен к регистру, поэтому ProCat и procat — разные переменные. Хорошей практикой считается давать название переменный с маленькой буквы, а если оно состоит из двух слов, каждое новое слово начинать с заглавной.

Например:

высота картины — imageHeight

номер комнаты — roomNumber

3.Давать осмысленное имя. Это облегчит дальнейшую работу и вам самим, и коллегам.

Например:

возраст собаки — dogAge

количество колёс в машине — carWheelCount

сумма налогов — taxSum

4. Воспользуйтесь различными онлайн-валидаторами имени переменной JavaScript. Например, на этом сайте можно проверить, нет ли проблем с использованием выбранного имени.

5. Константы принято писать заглавными буквами, разделяя нижним подчеркиванием.

Например: НДС

var TAX_RATE = 0.18

Как объявить переменную

Начнём с устаревшей версии, которая существовала в JS с самой первой версии. Остальные способы появились в стандарте ES6/ES2015.

var

var a = 10;

var a = "string";

var a;

Инициализация переменной значением является необязательной.

Очень часто начинающие разработчики думают, что слова undefined and undeclared — синонимы (не определена и не объявлена). Однако в JS это совершенно две разные концепции.

Свою роль в непонимание вносит сообщение в ошибке «is not defined». На самом деле там должно было бы написано «не определена» или «не найдена».

var a;

a: //undefined

b: //ReferenceError: b is not defined

Переменные в JavaScript не имеют типа: переменной может быть присвоено значение любого типа и позднее этой же переменной может быть присвоено значение другого типа. Это и есть динамическая типизация. Вам не нужно указывать тип данных переменной, когда вы её объявляете, типы данных преобразуются автоматически по мере необходимости во время выполнения скрипта в момент присваивания значения.

Важно отметить одну характерную особенность примитивных типов данных.

Примитивные типы используют семантику значения (значение копируется во время присваивания).

var a = 10;

var b = a;

a === b //true;

Можно взглянуть немного под другим углом: как переменные, которые содержат примитивный тип, сравниваются. Они сравниваются по значению: две переменные считаются одинаковыми, если они имеют одно и то же значение. Для чисел, логических значений, null и undefined это выглядит очевидным: нет никакого другого способа их сравнить. Однако для строк это утверждение не выглядит так. При сравнении двух строковых значений JavaScript считает их одинаковыми тогда и только тогда, когда они имеют одинаковую длину и содержат одинаковые символы в соответствующих позициях.

Объект использует семантику ссылки (присваивание формирует две ссылки на один и тот же объект).

let и const

В стандарте ES6/ES2015 было анонсировано ещё 2 способа задания переменной — используя ключевые слова let и const. На сегодня именно эти операторы рекомендуются к использованию (поэтому избегайте var!)

Отличия между let и const в том, что переменная, записанная через let, может быть изменена (будь то объект, или примитив).

Если переменная, записанная через const, примитив — её значение не может быть изменено. Если это объект, то свойства объекта могут быть изменены, а сам объект — нет.

Работа с примитивами

let a = 10;

undefined

const b = 10;

undefined

a = 11;

11

b = 11

Uncaught TypeError: Assignment to constant variable.

Работа с объектами

let a = { prop: "prop" };

undefined

const b = { prop: "prop" };

undefined

a.prop = "property";

"property"

b.prop = "property";

"property"

a = 'string';

"string"

b = 'string';

Uncaught TypeError: Assignment to constant variable.

Оператор Typeof

Ввиду того, что в JavaScript типизация динамическая, возникают такие ситуации, когда разработчик не уверен, какой именно тип данных хранится в переменной.

Чтобы не допустить ошибок и принять верное решение относительно дальнейших действий, может пригодиться оператор typeof, с помощью которого можно моментально узнать о типе данных.

Оператор возвращает строку с названием типа, применяется следующим образом:

let a = 10;

undefined

typeof a;

"number"

a = '10';

""10""

typeof a

""string""

let a = 10;

"undefined"

typeof(typeof a);

typeof(typeof a);

С примитивными типами данных оператор работает предсказуемо, но есть некоторые особенности, с которыми мы столкнёмся, когда будем изучать объекты.
