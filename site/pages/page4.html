1. Строки. Числа

В этом юните мы отдельно разберём такие типы данных, как строки (string) и числа (numbers). Начнём со строк.

Строка — это неизменяемая, упорядоченная последовательность 16-битных значений, каждое из которых обычно представляет символ Юникода. Строки являются типом данных, используемым для представления текста.

В некоторых языках программирования есть специальный тип данных для одного символа. Например, в языке С# это char. А в JavaScript есть только тип «строка» — string. Для представления единственного 16-битного значения просто используется строка с длиной, равной 1.

Строковый литерал — это ноль или более символов, заключенных в двойные ( " ) или одинарные ( ' ) кавычки. Строка должна быть ограничена кавычками одного типа, то есть либо обе одинарные, либо обе двойные.

Пример объявления строки:

let str = "1234";

Строки можно записывать в трёх вариантах кавычек — одинарных ', двойных " и обратных .

Во всех случаях строки будут прочитаны одинаково, не имеет значения, какие кавычки вы используете.

Важно помнить, что если вам потребуется кавычка в строке, то её нужно экранировать — используя символ \ перед ней:

let str = "I"m student"; // Syntax error

let str = "I\"m student";

Но если вы используете разные кавычки, то это не обязательно:

let str = "I’m student";

Работа со строками

Со строками можно делать много интересных вещей. Для этого существует очень много методов. Рассмотрим самые популярные.

Начнём с интересной особенности работы со строками, которые записаны в обратные кавычки. Такие строки называют шаблонными, а обратные кавычки называют оператором интерполяции.

1. Используя обратные кавычки как оператор интерполяции, мы можем писать код прямо в строке. Так интерпретатор выполнит код, как только его встретит. Для этого стоит поместить нужный код внутрь вот такой конструкции — ${ code }.

let firstName= 'Ivan';

let result = `${firstName} glad to see you!`;

console.log(result);

Ivan glad to see you!

2. Шаблонные строки позволяют задать так называемый теговый шаблон перед первой обратной кавычкой. Теговый шаблон — это функция, которая позволяет «разбирать» переданную строку на части и составлять из них другую строку.

Синтаксис метода: tag `string`.

Функция tag (название может быть любым) вызовется автоматически и получит в качестве аргументов строку и встроенные в неё выражения, чтобы их обработать. Подробнее об этом можно прочитать в документации.

Пример

let carName = 'Porsche';

let carPrice = 1000000;

function getCarPrice(strings, carNameExp, carPriceExp) {

let str0 = strings[0]; // "This car is "
let str1 = strings[1]; // " and it is "
let carPriceText = carPriceExp > 50000 ? 'expensive' : 'cheap'
return `${str0}${carNameExp}${str1}${carPriceText}`;

}

let result = getCarPrice`This car is ${carName} and it is ${carPrice}`;

console.log(result); // "This car is Porsche and it is expensive"

В качестве первого аргумента функция getCarPrice получает массив strings — это элементы строки, разбитые на части выражениями ${carName} и ${carPrice}. Соответственно, первым элементом массива будет строка "This car is ", а вторым — " and it is ".

Данная функция также принимает в качестве аргументов значения выражений внутри шаблонной строки, в нашем случае это название машины (carNameExp) и цена машины (carPriceExp). Название машины функция оставляет без изменений, а вот цену заменяет на определение «дорогая» или «дешёвая», в зависимости от значения цены. В итоге мы получаем следующую строку: "This car is Porsche and it is expensive".

Шаблонные строки очень удобны тем, что могут легко заменить операцию конкатенации (сложения) строк.

Оператор конкатенации — обычный бинарный плюс.

Когда движок встречает операцию сложения, в которой один из операндов строка, код выполняется как операция конкатенации и другой операнд превращает в строку.

Помимо сложения строк, часто возникает задача узнать длину строки. Для этого можно использовать свойство length, оно является встроенным.

let str= 'Count';

console.log( str.length );

5

Также может понадобиться перевести строку в верхний/нижний регистр. Для этого используйте методы .toLowerCase() и .toUpperCase():

let str= 'Count';

console.log( str.toLowerCase() );

count

console.log( str.toUpperCase() );

COUNT

Когда хотите получить подстроку, воспользуйтесь одним из 3-x методов: substring, substr и slice. Они все корректно выполняют задачу. Но формально у метода substr есть небольшой недостаток: он описан не в собственно спецификации JavaScript, и он может не работать в другом окружении, отличном от браузера. Однако на практике он работает везде.

Советуем обратить внимание на slice — он имеет больше «настроек», что позволяет ему быть более гибким, а также меньшее количество символов. Если научитесь обращаться с ним, этого метода будет достаточно.

Синтаксис метода: str.slice(start [, end]), где [, end] — необязательная часть и ее можно опустить, передав только стартовую позицию. Метод возвращает часть строки от start до (не включая) end.

let str = "string";

console.log( str.slice(0, 4) );

stri

Результат в консоли: 'str' — символы от 0 до 4 (не включая 4 (счёт идет от 0)).

Есть ещё очень много полезных методов, с которыми вы можете ознакомиться в спецификации.

Числа

Number — числовой тип данных в формате 64-битного. Их часто называют «числа двойной точности с плавающей точкой*» (стандарт IEEE 754) («double precision»).

*В каких морях ловить плавающую точку?

Термин «плавающая точка» — это прямой перевод с английского (floating point). В отличие от России, где целая часть отделяется от дробной запятой, в других странах при записи чисел используют точку. В настоящее время в русскоязычной литературе и технической документации можно встретить оба варианта: плавающая точка и плавающая запятая.

Данный тип включает как целые числа, так и дробные десятичные числа. В отличие от многих популярных языков, в JS нет отдельного типа для целых чисел, что является давним предметом критики. Числовые литералы чаще всего используются в виде десятичного числа, например:

let a = 42;

let b = 42.3;

Если целая часть равняется нулю, то её можно опустить и записать:

let a = 0.42;

let b = .42;

Есть одна особенность при работе с числами с плавающей точкой, которую необходимо знать:

0.1 + 0.2 === 0.3;

false

Несмотря на то, что результат должен быть true, мы получаем false, так как дробные числа хранятся в памяти с некоторой погрешностью, вследствие чего:

0.1 + 0.2 === 0.30000000000000004;

True

Максимальное безопасное целочисленное значение — Number.MAX_VALUE.

Существуют специальные числовые значения Infinity (как плюс, так и минус бесконечность) и NaN (не число).

Тип данных BigInt

Совсем недавно в спецификации появился ещё один тип данных — BigInt. Он ещё не вошёл в широкий обиход разработчиков, так как используется с очень большими числами, но нужно знать о его существовании и специфике.

Из спецификации:

BigInt — это встроенный объект, который предоставляет способ представлять целые числа больше 2 53 - 1, наибольшего числа, которое JavaScript может надёжно представить с Number примитивом. Это максимальное значение можно получить, обратившись к Number.MAX_SAFE_INTEGER.

Он похож на Number, но отличается в некоторых ключевых моментах: его нельзя использовать с математическими методами во встроенном объекте Math (чуть дальше поговорим об этом) и нельзя смешивать в операциях с любыми экземплярами Number.

Объект Math

Как правило, с числами мы привыкли делать различные математические действия. Например, простые — делить, складывать, умножать. И сложные — возводить в степень, округлять, находить логарифм от числа.

С простыми операциями всё понятно — умножаем, используя оператор *, складываем через +, вычитаем через -, делим, используя / или % (получаем остаток от деления).

Для сложных операций нет специальных операторов, но есть встроенный объект Math, хранящий в своих свойствах и методах различные математические константы и функции, которыми можно воспользоваться для вычислений.

Так, например, если нужно получить логарифм числа Пи, можно поступить следующим образом:

const result = Math.log(Math.PI);

console.log(result);

1.1447298858494002
